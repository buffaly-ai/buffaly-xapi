# Error Log

- `403 Unsupported Authentication` occurred when running `me`/`timeline` with only app-only bearer credentials; the CLI attempted bearer fallback for endpoints that require user-context auth. Fixed by enforcing user-context auth for `me`, `timeline`, `mytweets`, `mentions`, and `post` in `XClient`, with a clear configuration error that lists missing OAuth 1.0a fields (`access token`, `access token secret`, etc.). General lesson: never silently downgrade auth mode for endpoints with stricter auth requirements; fail fast with actionable missing-credential details.
- HTTPS calls failed with a generic `The SSL connection could not be established` message, which hid the actionable root cause (`SEC_E_NO_CREDENTIALS` from Schannel). Fixed by adding `HttpRequestErrorFormatter` in the CLI and using it in `Program` to include inner exception details plus targeted TLS guidance for Schannel credential failures. General lesson: preserve nested transport exception context in user-facing CLI errors so environment-level TLS/proxy issues can be diagnosed quickly.
