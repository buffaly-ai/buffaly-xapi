# Contributing

Buffaly is developed by Matt Furnari and published through the `buffaly-ai` GitHub organization.

## License for contributions

Unless otherwise agreed in writing, contributions to this repository are provided under the same license as the repository: GPLv3.

## CLA expectation

Before accepting major external code contributions, the project expects contributors to complete a Contributor License Agreement (CLA). This helps preserve the option to offer commercial licensing while keeping Buffaly core GPLv3 by default.

For now, please open an issue before sending substantial pull requests so licensing, scope, and review expectations can be confirmed.

## What belongs here

Core runtime code, typed actions, semantic entity/runtime infrastructure, plugin/tool examples, documentation, and source intended for inspection/debugging are in scope.

Domain packs, healthcare workflows, customer-specific connectors, deployment assets, implementation playbooks, sensitive demos/data, PHI, credentials, secrets, and confidential customer data are not in scope for public contribution.

Do not include PHI, credentials, secrets, customer data, or confidential information in issues, pull requests, examples, logs, screenshots, or test fixtures.
